cmake-node

发表于 2021-06-04 阅读次数：
  • cmake路径只支持/，不支持\，可以通过string(REPLACE "/" "\\" GROUP_PATH "${SRC_PATH_REL}")替换。
  • 如果cuda没有安装在默认位置，通过CMAKE_CUDA_COMPILER指定nvcc路径，set(CMAKE_CUDA_COMPILER "C:/UserProgramFiles/Cuda/bin/nvcc.exe")
  • cmake设置平台cmake -G "Visual Studio 15 2017" -A x64或者cmake -G "Unix Makefiles" -A Win32，然后可以设置Release模型，cmake --build . --config Release
  • linux可以通过LD_LIBRARY_PATH宏设置库搜索路径，而win没有相应的宏，为了让应用程序链接指定的库，有两种解决方法，请参考
    • [✓] 将库拷贝到应用程序目录
    • [✗] 将库路径加入PATH，并放在系统搜索路径前。(这个方法不一定可行，因为系统搜索路径在前)

  • 命令行编译.sln解决方法，

    • devenv demo.sln /Build
    • msbuild demo.sln /t:<proj1>[,<proj2> ...] /p:configuration="release"
    • cmake生成的*.sln 文件，解决方案中不包括INSTALL和ZERO_CHECK，ZERO_CHECK工程主要是监视CMakeLists.txt文件的变化。

  • cmake配置ninja生成器，首先需要将rc.exe, ninja.exe路径包含到PATH，不然ninja会出错，请参考。

  • cmake指定安装目录cmake .. -G "xxx" -DCMAKE_INSTALL_PREFIX=/install/dir
  • cmake构造指定目标cmake --build . --target demo
  • 安装项目，首先指定安装目录，cmake .. -G "xxx" -DCMAKE_INSTALL_PREFIX=/install/dir，然后有两种方式安装
    . msbuild /P:Configuration=Release INSTALL.vcxproj
    . cmake --build . --target INSTALL --config Release
  • cmake install指定安装目录cmake --install dir_to_cmake_install.cmake --prefix /usr，请参考
  • make test和make check用法，在cmake中添加add_test，然后用过cmake --build . --target check调用，需要把所有的测试放入all_demo中，请参考 add_test用法
    更简单的用法是直接在add_subdirectory(src)前添加enable_testing()，然后add_test(demo, demo)，然后用ctest调用测试。ctest运行指定测试，ctest -R -V demo, ctest -N查看所有测试

  • cmake设置项目属性

    win 构建工程

    cmake通常有两种构建工程的方法，
    . 生成makefile, 通过make(nmake)构建。可以通过cmake .. -G "Unix Makefiles"指定生成Makefile，或者在CMakeLists.txt的同一级创建一个PreLoad.cmakeset(CMAKE_GENERATOR "Unix Makefiles" CACHE INTERNAL "" FORCE)。请参考

将c++编译为python模块时找不到编译器，报错Unable to find vcvarsall.bat。参考博文解决。找到python的distutils/msvc9compiler.py文件，做两处修改，

  • get_build_version（第172行），添加return 9.0
  • find_vcvarsall（第222行），添加return r’你的Microsoft Visual C++ Compiler for Python路径’（return r'D:\software\visualStudio\VC\Auxiliary\Build\vcvarsall.bat'）

此外也可以修改VS90COMNTOOLS环境变量解决，请参考。
