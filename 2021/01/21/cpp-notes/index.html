Cpp-Notes

发表于 2021-01-21 更新于 2021-06-04 分类于 编程 阅读次数：
  • 在常函数中可以修改mutable关键字修饰的成员变量。
  • dynamic_pointer_cast出错会返回nullptr。
  • c++通过条件编译来避免头文件被多次包含。在实际开发中，直接把另一个文件拷贝成当前文件，并在上面做修改。如果忘记修改预处理宏，则会导致文件无法被正确包含的编译出错。正确做法是拷贝文件后因立即修改预处理宏，或者使用#pragma once来避免这个问题。
  • 类的初始化列表不能调函数
  • error C2589: “(”: “::”右边的非法标记 。在VC++种同时包含头文件#include 和#include 后就会出现无法正常使用std标准库中的min和max模板函数。在预定义处理器中添加定义NOMINMAX来禁止使用Vsual C++的min/max宏定义。请参考。
  • fbxsdk未定义，需要添加FBXSDK_SHARED宏。If you are using a dynamic library version of the FBX SDK, add FBXSDK_SHARED to the preprocessor definitions of your project。You need to define “FBXSDK_SHARED” if you are using the DLLs.
  • 在模板类中，可以使用模板提供的类型，如typedef typename TDataType::Real Real，其中TDataType是模板参数，而Real是TDataType中定义的类型（结构体或者类）。
  • 生成shared_ptr方法，
    • shared_ptr t_ptr = new T(t);
    • shared_ptr t_ptr = make_shared(t);(传入构造函数参数)
  • 用父类数据给子类初始化，通过父类拷贝构造函数，这样子类和父类数据就相等了

    多线程

  • 显式指定子类调用父类方法，Child.Parent::method()，对于模板类不用加模板参数。
  • vector<set<int>>不支持并行写入，set插入元素可能导致vector reallocate
  • 多线程运行结果和关闭多线程运行结果不一致，可能是浮点数相加的前后顺序造成的。
  • openmp设置线程数
    • omp_set_num_threads()设置全局线程数
    • #pragma omp parallel for num_threads(2*numProcs-1)单次设置线程数
    • export OMP_NUM_THREADS = 3通过环境变量设置线程数
    • omp_get_num_procs()得到机器核心数

    Gtest

  • gtest用法，下载，编译，安装到指定位置cmake --install dir_to_cmake_install.cmake --prefix /usr。
    包含头文件，链接gtest和gtest_main库。

  • 编译gtest时，可以选择设置/MD或者/MT。如果项目中使用/MD，则应该使用/MD编译；如果项目中使用/MT，则应该使用/MT编译。
  • gtest和gtest_main的区别，gtest_main中包含一个main函数，会注册所有test
  • 单元测试
  • gtest
ASSERT_LT(<) ASSERT_LE(<=) ASSERT_GT(>) ASSERT_GE(>=)
ASSERT_TRUE(1) ASSERT_FALSE(0) ASSERT_EQ(==) ASSERT_NE(!=)
  • gtest在TEST()测试套件外使用ASSERT_*， 该函数必须返回void（ASSERT宏返回空值）。
  • TEST_F可以测试有相同流程的程序，但是每个test case都会运行setup
