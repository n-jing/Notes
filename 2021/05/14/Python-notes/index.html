Python调用C++模块

  • 将c++编译为python模块供python调用。主要流程为，请参考

    • 将c++编译为lib
    • 编写swig的*.i文件
    • 配置setup.py，包括include_dirs、library_dirs、libraries、extra_compile_args等。
    • 编译

  • 如模块为纯c++编写，不涉及与python的数据交互（如模块调用没有参数，或者参数只是字符串，浮点数等），则相对容易一些，只用编写好*.i文件即可。demo.c_method(3, "file_path", 0.1)

  • 如模块有与python的数据额交互，则需要包含相应的头文件，注册函数以及初始化模块。demo.c_method([1, 2, 0.5])，请参考

    • #include<Python.h>
    • 编写调用接口。
      • 处理从 Python 传入的参数。
      • 实现功能逻辑。
      • 处理 C 中的返回值，包装成 Python 对象。
    • 在一个 PyMethodDef 结构体中注册需要的函数。
    • 在一个初始化方法中注册模块名。
    • 最后，通过swig将文件编译为*.pyd。

PyMethodDef中注册函数

将c++编译为python模块时找不到编译器，报错Unable to find vcvarsall.bat。参考博文解决。找到python的distutils/msvc9compiler.py文件，做两处修改，

  • get_build_version（第172行），添加return 9.0
  • find_vcvarsall（第222行），添加return r’你的Microsoft Visual C++ Compiler for Python路径’（return r'D:\software\visualStudio\VC\Auxiliary\Build\vcvarsall.bat'）

此外也可以修改VS90COMNTOOLS环境变量解决，请参考。

python常见用法

  • 常见用法
  • ‘’（单引号）和””（双引号）用法相同，’’（单引号）中”（双引号）不需要转义，””（双引号）中’（单引号）不需要转义，这样让程序更加简明；’’’’’’（三引号）主要是方便分段。
  • from termcolor import colored，termcolor可以对输出进行颜色管理print(colored('[ error ]: ', 'red'), "error")
  • with ... as ...，with适用于对资源进行访问的场合，确保不管使用过程中是否发生异常都会执行必要的“清理”操作，释放资源，比如文件使用后自动关闭／线程中锁的自动获取和释放等。请参考
  • python[关键字]（https://zhuanlan.zhihu.com/p/350295664）
  • yeild返回一个可迭代的 generator（生成器）对象，可以使用for循环或者调用next()方法遍历生成器对象来提取结果。与返回list相比，占用内存更小，但是多次访问存在重复计算的问题。
  • 异常处理，try/except，try/except...else，try-finally，请参考
